<!-- Policy Related Table (Using LangChain Parent Document Retriver)-->
2. policies (Static Rules Table)
* policy_id (PK)
* alias (VARCHAR) - Free Name given by HR 
* title (VARCHAR) - e.g., 'Business Travel Policy'.
* reimbursable_category (LIST) - More specify on what can be reimbursement e.g., [Air Transportation, Meals, Accommodations].
* effective_date (TIMESTAMPZ) - e.g., '2024-07-01'.
* overview_summary (TEXT) - The LLM-generated summary for broad queries.
* mandatory_conditions (TEXT) - The SOP checklist text utilized by the RAG system. (will turn into force json output for agent)
* source_file_url (VARCHAR) - Path to where the original PDF is saved 
* created_at (TIMESTAMPZ)
* status (VARCHAR) - "IMPENDING", "ACTIVE", "EXPIRED" (Don't remove un-used policy as past reimbursment may refer to them)

3. policy_sections (The Unified Vector Table)
This table is the splitted policy and appendix, used for RAG purposes 
* section_id (UUID, PK): Unique identifier for the chunk.
* policy_id (FK -> policies.policy_id): Links the rule back to the specific policy. 
* content: The actual Markdown text (e.g., the procedures for international travel). 
* embedding: The VECTOR column for pgvector similarity searches.
* metadata (JSONB): Stores structural context to help the LLM cite its sources.
    * chunk_index 
    * headers: {"H1": "5. BUSINESS TRAVEL PROCEDURES", "H2": "a. Approval"} 
    * page_no: The page number from the PDF (e.g., 6). 
    * is_appendix: A boolean flag to indicate if this chunk is from the main policy or a supporting document like Appendix 1.

<!-- Reimbursement Related Table -->
4. reimbursements (Core Transaction Table)
* reim_id (PK)
* user_id (FK) -> Refers to employees.User_ID
* policy_id (FK -> policies.policy_id)
* main_category
* sub_category
* employee_department (VARCHAR) - (snapshot from employees table when applying reimbursement)
* employee_rank (INT) - (snapshot from employees table when applying reimbursement)
* currency (VARCHAR) - e.g., RM, USD, IDR
* amount (DECIMAL)
* judgment (VARCHAR) - Stores the final output: "APPROVE" or "FLAG" or "MANUAL REVIEW".
* status (VARCHAR) - Change by HR (e.g: "REJECTED", "REVIEW" (Default, AI always set this regardless), "APPROVED" (Approved by HR, reimbursement money pending), "PAID" (reimbursement have done))
* chain_of_thought (JSONB) - As describe as step above this
* summary (TEXT) - The overall reasoning generated by the combination of related claims_audit_logs.reason.
* created_at (TIMESTAMPZ)
* updated_at (TIMESTAMPZ)

5. supporting_documents (documents uploaded by employees to claim reimbursement)
* document_id (PK)
* reim_id (FK) - NULLABLE when staging, will be updated when Reimbursement claim is submitted.
* user_id (FK)
* name (TEXT) - name of the document
* path (VARCHAR) - (Note: Seperate folder for each user_id)
* type (VARCHAR) - (e.g. PDF or Image)
* is_main - A boolean flag to identify whether the document uploaded is the main receipt (which cannot be duplicate)
* extracted_data (JSONB) - free from JSON with each document_class have it's own template.
* created_at (TIMESTAMPZ)
